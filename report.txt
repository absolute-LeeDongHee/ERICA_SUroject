2019/12/18
뭐하지 뭐하지 계속 고민하면서 여러가지 건드려보다가 dialogflow라는 것을 발견했다.
쉽게 챗봇을 만들 수 있도록 구글에서 무료로 서비스를 제공하고 있었다.
인사 기능을 만들어봤는데 동록한 입력 값과 동일하지 않고 비슷하다면 그에 맞는 대답을 했다.
그래서 이를 이용하여 롤 프로게이머 관전 영상을 제공하는 챗봇을 만들면 어떨까 생각했고
dialogflow에 일부 선수들의 활동명이 담긴 데이터베이스를 만들었고
이를 통해 선수 이름을 입력 받는 기능까지 구현했다.

2019/12/19
선수 이름을 입력 받으면 해당 선수의 실시간 관전 영상을 제공하려고 방법들을 찾아봤다.
그런데 맙소사.. 실시간 관전을 위해서는 라이엇 게임즈의 api를 써야 하는데 라이엇 게임즈가
내년 1월 6일까지 api 허가를 막았다. 그리고 챗봇을 통해 입력 받고 입력 값에 해당하는 관전
영상을 제공하는 기능을 구현하기에는 내가 모르는게 너무 많았다. 어떻게 방법이 없을까
계속 찾아보느라 커밋도 못했다...

2019/12/20
내 노트북이 우분투를 실행을 못해서  오픈 소스 프로젝트는 집에 있는 컴퓨터로 진행하는데
저번에 수업시간에 알려주신 다른 방법들을 사용하면 되지 않을까해서 처음에는 WSL을 설치하려했다.
하지만 이 대단한 노트북은 Linux용 Windows 하위 시스템 기능이 없었다. 그래서 goorm.ide로
dialogflow에서 추가한 기능들을 올리고 github를 봤더니 commit이 초기화가 되었다;;
그리고 여젼히 챗봇을 통해 어떻게 관전 영상을 제공할 지 해답을 찾지 못하고 있다.

2019/12/21
관전 기능의 해결 방안으로 실시간 관전 사이트의 주소를 알려주는 것을 생각했다.
막상 만들고 보니까 챗봇이 너무 부실했다. 그래서 전적 검색 기능을 넣으려고 했는데
찾아보니 전적 이외에도 다른 기능들을 넣을 수 있을 것 같았다. 웹에 있는 데이터들을
파이썬으로 어떻게 가져올 수 있을까 방법을 찾아봤는데 웹 크롤링이라는 것이 있었다!!!
이거라면 웹에 나와있는 데이터들을 모을 수 있겠구나 해서 예제 코드들을 보면서 크롤링 파일을
만들어 보았다. 해답을 찾아서 기뻤는데... 여전히 dialogflow에서 입력 받고
이를 처리하는 방법을 모르겠다. 설상가상으로 원래 dialogflow와 연동하려 했던
카카오톡 플러스 친구 챗봇도 개발자 승인을 받아야 했다... 망했다. 그냥 챗봇은 포기해야겠다.
아니 근데 전적 사이트 크롤링은 쉬운데 다른 데이터는 크롤링하기 왜이리 힘든거야;; 태그가
여려개가 있는 데이터는 원하는 데이터만 가져오기가 쉽지 않다.

2019/12/20
챗봇을 포기하고 계속되는 삽질 끝에 다른 데이터도 크롤링 할 수 있는 파일을 만들었다. 그리고
자고 일어났는데 생각해보니 디스코드 챗봇은 만들 수 있을 것 같았다. 그래서 개발자 홈페이지에
들어갔더니 다행히 승인 없이도 챗봇을 만들 수 있었다. 문제는 파일을 어떻게 챗봇 기능으로
구현할 방법이었다. 챗봇으로 닉네임을 입력 받으면 그걸 어떻게 입력 값으로 줘서 크롤링을 해야할까?
그래서 나는 해답을 찾기 위해 어제에 이어 오늘도 삽질을 했다. 근데 생각해보니 커뮤니티가 있는데
왜 나는 여태까지 삽질을 하고 있었던걸까? 바로 질문했더니 사용자가 챗봇에게 입력하는 것은 문자열이니
배열처럼 시작 위치를 설정하면 그것을 크롤링할 입력 값으로 받아올 수 있다는 답을 받았다.
오! 이렇게 입력 값을 받아서 실행해보니 잘 된다ㅠㅠ 고마워요 커뮤니티!!
